# Protected Branches

Protected branches are a feature of platforms like GitHub and GitLab that allow you to enforce rules on your branches.

You can use protected branches to:

*   **Prevent direct pushes to the branch:** All changes must be made through a pull request.
*   **Require status checks to pass before merging:** This can be used to run automated tests.
*   **Require code review before merging:** Ensure that at least one other person has reviewed the code.
*   **Prevent force pushes:** This protects the branch's history.

Protected branches are a crucial tool for maintaining a high-quality and stable codebase, especially in a collaborative environment. It's a best practice to protect your `main` branch.
